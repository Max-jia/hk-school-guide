# Blog 寫作與流量優化規則（2026年7月版）

> 核心目標：提升網站自然搜尋流量。以下規則基於 Ahrefs、Backlinko、Serpstat 等 SEO 權威來源的數據驗證，以及 188 篇 Blog 的實戰 A/B 測試結果。

---

## 一、選題策略（流量導向）

### 1.1 關鍵字優先級（按 ROI 排序）
| 優先級 | 類型 | 示例 | ROI 依據 |
|--------|------|------|---------|
| 🔴 最高 | **對比/替代類** | 「聖若瑟 vs 喇沙」「官津 vs 直資」 | 對比內容點擊率是普通 SEO 指南的 **3 倍**（23 vs 8 clicks/post） |
| 🟡 高 | **Google Trends Breakout** | 搜索量突然飆升的學校名 | 搶佔上升期流量紅利，競爭低 |
| 🟢 中 | **長尾問題型** | 「41校網房租多少」「細B考名幼攻略」 | 長尾詞佔總搜索流量 70%+，單個競爭低 |
| 🔵 補充 | **排名/榜單型** | 「2026 香港小學排名」 | 搜索量大但競爭高，已有 HKPEP/Kidemy 佔據 |

### 1.2 簡繁雙關鍵字策略
- Google 對簡體和繁體是**獨立索引**的
- 同一主題須覆蓋簡繁兩套關鍵字
- 例如：「香港小学排名」和「香港小學排名」要分別做文章
- 用 Google Trends 分開查簡繁搜尋趨勢，挑各自熱門詞

### 1.3 主題集群（Topic Cluster）策略
不要孤立寫單篇文章。圍繞核心主題建立 **支柱頁 + 集群頁**：

```
支柱頁：香港小學擇校終極指南
├── 官津 vs 直資 vs 私立（對比集群）
├── 41校網 vs 12校網（對比集群）
├── 一條龍/直屬/聯繫詳解（概念集群）
├── 聖若瑟小學深度拆解（單校集群）
├── 港大同學會小學深度拆解（單校集群）
└── 陳守仁小學深度拆解（單校集群）
```

每篇集群頁 → 鏈回支柱頁（1 個 pillar link）+ 鏈到 3 個姊妹集群頁（contextual links）

## 二、標題與 Meta 優化

### 2.1 標題公式（提升點擊率）
| 公式 | 示例 | CTR 提升依據 |
|------|------|-------------|
| 數字 + 情緒詞 + 反套路 | 「我親測了5類香港小學，新港人選這類最划算」 | 數字標題 CTR +36%（Backlinko 研究） |
| 對比型 | 「聖若瑟 vs 喇沙：灣仔12校網 vs 九龍塘41校網男校終極對比」 | 對比內容點擊率 3x |
| 年份 + 榜單 | 「2026香港小學排名Top50」 | 年份關鍵字搜尋意圖強 |

### 2.2 Meta Description 規則
- 150 字以內
- 含目標關鍵字（Google 會加粗匹配詞）
- 用「數字 + 問題 + 解決方案」結構
- 低 CTR 頁面優先修復（CTR < 1% 但展示量 > 1000 → 立即改）

## 三、內容結構（AI 時代雙重優化）

### 3.1 同時服務兩個讀者：人類 + AI 引擎
2026 年 ~1.6% 流量來自 ChatGPT/Claude 等 AI 搜尋，但 AI 流量**轉換率是 Google 的 5 倍**（14-17% vs 2.8%）。需同時優化：

| 優化目標 | 做法 |
|---------|------|
| **人類讀者** | 爆款風格（第一人稱 + 反套路 + 表格 + 抄作業公式） |
| **AI 提取** | 每 150-200 字嵌入一個「引用鉤子」：一句話定義、數據塊、迷你 FAQ |
| **段落長度** | ≤ 80 字/段，方便 AI 擷取 |
| **FAQ 結構** | 用 Q&A 格式寫核心段落，LLM 可直接當答案引用 |

### 3.2 內容結構模板
```
H1 標題：數字 + 情緒詞 + 關鍵字
開頭：個人故事/數據衝擊（3-4 句，含目標關鍵字）
H2：核心數據速查（表格）
H2：深度拆解（每段 ≤80 字，含引用鉤子）
H2：風險/陷阱（warn box）
H2：適合誰？（highlight box + FAQ 格式）
CTA：自然引導到產品頁/報告頁
來源註釋
```

### 3.3 EEAT 信號（Google 排名必要條件）
- **Experience**：第一人稱視角、踩坑記錄（不是百科語氣）
- **Expertise**：引用本平台四因子數據、教育局官方數據
- **Authoritativeness**：Blog 首頁有作者介紹、網站有關於我們頁面
- **Trustworthiness**：每篇文章標註更新日期、數據來源可驗證

## 四、寫作規則

### 4.1 爆款風格
- ✅ 第一人稱、個人視角
- ✅ 反套路開頭（「最開始我也…後來發現全錯了」）
- ✅ 結尾「抄作業公式」
- ✅ 數據衝擊（7000+ 搶 160 個學位）
- ❌ 百科語氣、中立報導

### 4.2 去 AI 味（humanizer-zh）
| 刪除 | 替代 |
|------|------|
| 「值得一提的是」「由此可見」「此外」 | 直接刪 |
| 「不僅…而且…」 | 拆成兩句 |
| 「這意味著」 | 「也就是說」 |
| 段末總結句 | 用公式/CTA 替代 |
| 三個一組的形容詞 | 只用最準確的一個 |

### 4.3 引用鉤子（AI 提取優化）
每 150-200 字放一個可被 AI 直接引用的內容塊：
- **一句話定義**：「一條龍 = 100% 直升，直屬 = 85% 保留，聯繫 = 25% 共享」
- **數據塊**：「全港 669 所小學，官立 33 所，資助 495 所，直資 22 所，私立 102 所」
- **迷你 FAQ**：「Q: 41校網房租多少？A: 兩房月租 $22,000-$35,000」

## 五、內鏈策略

### 5.1 每篇文章的內鏈規則
```
每篇 Blog 文章：
├── 1 個鏈到支柱頁（pillar link）
├── 3 個鏈到姊妹集群頁（contextual links）
└── 1 個 CTA 到產品頁（報告頁/匹配頁）
```

### 5.2 內鏈優化週期
- 新文章發布 → 從 3 篇舊文章加鏈到新文章
- 每季度掃描孤兒頁面（無內鏈指向的頁面）→ 補鏈接

---

## 六、內容維護（比發新文更重要）

### 6.1 數據刷新
- 每 90 天微更新（價格、排名、政策變動）
- 更新後在頁面頂部標註「更新日期」
- 團隊做內容刷新 sprint → **30 天內流量提升 30%**（Siteimprove 數據）

### 6.2 低 CTR 頁面優先修復
- 從 Search Console 找「展示量 > 1000、CTR < 1%」的頁面
- 優先改這些頁面的標題和 meta description
- 效果 > 寫 10 篇新文章

---

## 七、技術規則

- 簡繁雙版，互相鏈接
- 發布日期分散（每天 1-2 篇，不要集中）
- 用 IndexNow 推送新內容到 Bing（連帶覆蓋 ChatGPT Search）
- `<title>` 含目標關鍵字，`<meta description>` ≤150 字

---

## 八、禁止清單

- ❌ 孤立寫文章（無內鏈）
- ❌ 所有文章同一天發布
- ❌ 用 .json 檔名作為來源標註
- ❌ AI 填充詞（此外、值得一提的是、由此可見）
- ❌ 只用大詞不做長尾詞
- ❌ 只發新文不維護舊文
- ❌ 忽略簡繁雙關鍵字

---

## 九、發布檢查清單

- [ ] 選題有數據支撐（Google Trends / Search Console / 長尾關鍵字）
- [ ] 標題有數字/情緒詞/對比元素
- [ ] Meta description ≤150 字，含關鍵字
- [ ] 內容有引用鉤子（每 150-200 字）
- [ ] 內鏈：pillar link + 3 cluster links + CTA
- [ ] 簡繁雙版已生成
- [ ] 已過 humanizer 去 AI 味
- [ ] 日期與其他文章分散
- [ ] Blog 首頁索引已更新
- [ ] 從 3 篇舊文章加了鏈到新文章
