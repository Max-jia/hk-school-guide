# 深度择校报告生成规则（完整版）

---

## 一、报告文件结构

### 1.1 文件名
- 简体版：`report-{code}.html`（如 `report-srl.html`）
- 繁体版：`report-{code}-tc.html`（如 `report-srl-tc.html`）

### 1.2 HTML 必须包含（固定顺序）
```
<!DOCTYPE html>
<html lang="zh-HK" data-report-code="{code}">
<head>
  meta charset + viewport
  title ({校名} · 深度择校报告 | 港校指南)
  共享样式：style.css, report.css
  内联样式：score-bar / pros-cons / radar / decision-tree 等
  paywall.css
  Supabase SDK (CDN)
  auth.js
  paywall.js
</head>
<body>
  <header> (report-hero)
  <main>
    第 0 节：数据可靠性声明
    第 1 章
    付费墙 (paywall-overlay)
    <div id="premium-content">
      第 2-8 章
    </div>
    底部 CTA banner
  </main>
  <footer>
</body>
</html>
```

## 二、数据来源与引用规则

### 2.1 来源标注层级

| 用途 | 标注文字 | 可靠性 |
|------|---------|--------|
| 学校基础数据（名称/校网/类型/学费/性别/语言/班师比） | `✅ 教育局学校概览 2025` | 官方 ✅ |
| 升中政策（直属/联系/一条龙比例） | `✅ EDB 政策` | 官方 ✅ |
| 学校评级/四因子评分 | `本平台四因子评级` | 自研体系 |
| 中学 Banding | `🟡 综合公开数据评估（非官方）` | 多源综合 |
| 校方特定信息（校训/创校/校舍/设施等） | `🟡 校方官网` | 官方但非政府 ✅ |
| 媒体报道 | `🟡 {媒体名} {年份}` 如 `🟡 TOPick 2024` | 正规媒体 |
| 家长口碑/讨论 | `🔴 教育王国(BK)家长分享` | 匿名论坛 |
| 专家分析 | `💬 专家分析` | 主观 |

### 2.2 禁止的来源标注
- ❌ `.json` 文件名（如 `primary_tiers_v9.json`）
- ❌ 内部数据库名（如 `sec_school_banding_v2.json`）
- ❌ "本平台" 用于非评级/Banding 的事实
- ❌ 模糊来源（如 "综合"、"多方" 不加说明）
- ❌ 第三方聚合网站作为基础事实的唯一来源（如 schooland、SOSOMAMA）

### 2.3 「本平台」使用规则
- ✅ 可用于：四因子评级结果、中学 Banding 共识
- ❌ 不可用于：学校基础数据、政策、媒体报道、家长口碑

## 三、报告内容结构（8 章）

### 第 0 节：数据可靠性声明（固定模板，不可修改）
```html
<div class="callout warn">
  <strong>本报告数据可靠性声明</strong><br />
  本报告所有事实性数据已尽可能经过多方核实，并按核实程度标注：<br />
  · <strong>✅ 多方核实</strong>：政府公开数据(CHSC/EDB)或多个独立来源交叉确认<br />
  · <strong>🟡 单一可靠来源</strong>：正规媒体报道或官方信息，未经第二个来源交叉验证<br />
  · <strong>🔴 有限来源·仅供参考</strong>：家长讨论区/单一匿名帖等，可信度有限<br />
  · <strong>💬 主观分析</strong>：专家观点/分析，非事实陈述<br />
  所有客观数据一经发现更准确的信息源将立即更新。主观分析部分不构成入学建议。
</div>
```

### 第 1 章：学校定位与核心竞争力
- 档案表（≥8 行）：全名、地区/校网、等级、类型/性别、学费、班师比、教学语言、关联中学
- 每条标注数据来源（✅🟡🔴💬）
- 四因子评分条（4 条 bar 图）+ 读解文字
- 专家定位分析（💬 主观）：一句话定调 + 2-3 段深度分析
- 雷达图（SVG）：四个维度（升学保障、口碑共识、入读难度、校友网络）
- 适合/不适合的孩子类型（pros-cons 双列）

### 第 2 章：教学深度解读
- 课程体系详解（解读「为什么」，不是罗列）
- 考试与功课量
- 师资分析
- 校舍与设施
- ⚠️ 教学局限（≥3 条，诚实指出）

### 第 3 章：升中通路全景（小学）/ 升小通路全景（幼稚园）
- 联系/直属/一条龙体系说明
- 关联中学详情（含 Banding）
- 升中派位实际数据（有就写，没有标「校方未公开」）
- ⚠️ 风险提示（≥2 条）

### 第 4 章：入读攻略
- 录取难度总评
- 自行分配学位阶段（计分制/面试情况）
- 统一派位阶段（甲部/乙部/校网要求）
- 提升入读概率建议（≥5 条）
- 叩门攻略

### 第 5 章：插班与叩门攻略
- 插班概览（机会大小/最佳时机）
- 插班申请流程（表格：时间/材料/笔试/面试/标准）
- 求位信策略（时机/长度/结构/附件/禁忌）
- 必须针对该校写，不能用通用模板

### 第 6 章：在读家庭真实画像
- 口碑采样标注（平台/帖数/时间跨度/可信度声明）
- 👍 家长最满意的 5 点
- 👎 家长最不满的 5 点
- 「希望你入学前就知道」匿名分享（≥2 条）
- 每条必须标注来源

### 第 7 章：同类校横向对比
- ≥3 所同级别/同区竞品
- 一表对比（等级/类型/校网/学费/直属中学/入读难度/特色）
- 选择决策树（≥3 级分支）

### 第 8 章：专家总结与建议
- 价值判断（值不值？什么家庭适合？）
- 投入 vs 回报（成本估算）
- ⚠️ 风险提示
- Plan B 推荐（≥3 个替代方案，表格输出）
- 一句话行动建议

## 四、写作规则

### 4.1 必须遵守
- 所有报告只用**简体中文**撰写，学校官方名称保留繁体原文
- 不编造数据 — 无法核实的写「校方未公开」或「暂无数据」
- 不写绝对化语言（「一定」「100%」「足矣」「必进」）
- 诚实指出局限 — 第 2 章 ≥3 条，第 3 章 ≥2 条
- 每章末尾加 `📎 来源：{来源列表}` 脚注

### 4.2 禁止内容
- ❌ research 过程笔记（如 deep-research、babyeduhub、vote 0-3）
- ❌ 内部数据库文件名（.json）
- ❌ TODO / 待补充 占位符
- ❌ 跨校复制粘贴残留
- ❌ 章节标题重复
- ❌ 空表格/空列表
- ❌ 小红书/知乎/百度作为 HK 本地口碑唯一来源

## 七、繁体版规则

- 每份简体报告必须同步生成繁体版
- 繁体版仅文字转繁体中文，结构完全一致
- 文件名加 `-tc` 后缀
- Hero 的简繁切换链接指向对方版本
