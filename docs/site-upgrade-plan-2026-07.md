# 港校指南 · 网站升级方案 v1

> 日期：2026-07-15 | 基于《竞品深度研究报告》提出

---

## 核心思路

竞品分析告诉我们：**我们有三个竞品没有的独门武器（匹配引擎、8 章深度报告、付费墙模式），但有两个短板（Blog 内容不够暖、首页没有把独门武器推到最前面）。**

升级方案分三个阶段，每阶段做完可以独立上线，不需要等全部完成。

---

## 第一阶段：首页 + Blog 升级（预计 3–5 天）

### 1.1 首页首屏重构

**现在的问题**：首页 hero 区讲「智能匹配」，但匹配引擎那个表单藏得太深，用户一眼看不到。竞品（Schooland/Ohpama）都是把核心功能直接怼在首屏。

**改为**：
```
首屏分两个板块，上下排列：

┌─────────────────────────────────────┐
│  🎯 快速匹配（4 个下拉直接开始）      │
│  校网 ▾  预算 ▾  语言 ▾  路线 ▾  [开始]│
│  → 30 秒出结果，不用注册，完全免费     │
├─────────────────────────────────────┤
│  📚 深度择校报告（123 份·HK$198 全解锁）│
│  [S 级神校] [A+ 级名校] [A 级] [B 级]  │
│  最近新增：聖伯多祿/培基/王錦輝 →      │
└─────────────────────────────────────┘
```

**改动范围**：
- `index.html` + `index-tc.html` hero 区重构
- 把匹配表单从独立页搬到首页首屏（或做成一个显眼的入口卡片直跳）
- 底部加「最新报告」动态卡片（3–5 张，从 JS 数组取最新几条）

**参考**：Ohpama 首页直接展示核心搜索功能；Schooland 首页就是搜索框

### 1.2 Blog 内容风格升级

**现在的问题**：我们现有的 14 篇 blog 偏「资料型」——像百科条目，不太像「有温度的内容」。Ohpama 靠赵荣德+真实个案取胜。

**改为**：
- 每篇 blog 至少包含 **一个具体个案/场景**（不指名道姓，但要有画面感）
  - ❌ 「聖若瑟小學是灣仔區A級資助男校，聯繫Band 1A聖若瑟書院」
  - ✅ 「家住灣仔的陳太，兒子在中文幼稚園讀 K3，她在自行分配階段計算自己只有 15 分——天主教背景加 5 分、首名子女加 5 分、適齡加 10 分。她糾結該不該把 15 分押在聖若瑟…」
- **加专家署名**：每篇末尾标注「本文由港校指南教育研究团队撰写，数据来源：教育局 + 本平台核实数据库」
- **加「一句话送给家长」**：每篇结尾一段加粗的金句总结

**改动范围**：
- 不是重写 14 篇，而是给每篇加一个「场景引入段」（约 200 字）+ 结尾金句
- 后续新文章按新标准写

### 1.3 Blog 内容日历（按教育局收生节奏）

| 月份 | 主题 |
|------|------|
| 7–8 月 | 叩门攻略、派位放榜解读、暑假插班准备 |
| 9 月 | 自行分配学位策略、计分攻略、校网分析 |
| 10–11 月 | 直资/私立小学面试真题、面試NG位汇总 |
| 12–1 月 | 统一派位填表策略、甲部 vs 乙部选择 |
| 2–3 月 | 幼稚园 K1 面试准备、名幼对比 |
| 4–5 月 | 升中呈分试准备、一条龙 vs 直属详解 |
| 6 月 | 派位结果解读、叩门倒计时 |

**改动范围**：建立内容日历文档 `docs/content-calendar.md`，按月规划选题

## 第二阶段：面试题库 + 用户信任建设（预计 2–3 天）

### 2.1 面试题库页面

**竞品对标**：MySchool.hk 的核心流量来自「历年面试题目」——这是我们完全没有的板块。

**做**：
- 新建 `interview-questions.html`（简+繁）
- 从 BK/教育王国整理 20–30 所热门小学的面试形式和高频题目
- 按「面试形式（小组/个面/游戏）+ 高频题 + 校长原话」三个维度呈现
- **不做**需要逐校核实的深度面试攻略（太耗人力），做**轻量结构化题库**

**数据来源**：教育王国公开帖汇总（标注「🔴 来源：家长分享」）

### 2.2 「数据来源」信任页

**现在的问题**：我们对外说「数据源自教育局」，但用户看不到证据。竞品（Schooland）直接链到学校官网，透明度更高。

**做**：
- 新建 `data-sources.html`（简+繁）——一页讲清楚我们每个数据字段的来源
- 表格形式：「字段 | 来源 | 更新频率」
- 增强用户信任，降低「这数据靠谱吗」的疑虑

## 第三阶段：体验打磨 + 转化漏斗（预计 2–3 天）

### 3.1 匹配结果→深度报告 的流转链路

**现在的问题**：用户在匹配引擎里搜到一所学校 → 看到一个结果卡片 → 然后呢？没有明显的「了解更多」入口。

**改为**：
- 匹配结果卡片底部加 **「📖 查看完整深度报告（8 章）」** 按钮
- 如果该学校无报告 → 显示「🔔 暂无深度报告，点击此处告诉我们你想了解哪所学校」
- 打通「免费匹配 → 深度报告 → 付费转化」的漏斗

### 3.2 首页转化率提升

- 匹配引擎入口按钮改大、改色（现在偏小偏淡）
- 加一个小小的「已有 XXX 位家长使用了择校匹配」计数器（用 localStorage 模拟，不造假）
- 「深度择校报告」入口卡片加轮播（展示最新 3 份报告封面）

## 优先级排序

| 优先级 | 模块 | 原因 |
|--------|------|------|
| 🔴 P0 | 1.1 首页首屏重构 | 现在首屏浪费了，用户一眼看不到核心功能 |
| 🔴 P0 | 1.2 Blog 风格升级 | 现有 blog 偏冷，需加温度才能留住人 |
| 🟡 P1 | 2.1 面试题库 | MySchool 靠这个吸流量，我们可以快速抄作业 |
| 🟡 P1 | 1.3 内容日历 | 按收生节奏推文，SEO 效果好 |
| 🟢 P2 | 3.1 匹配→报告链路 | 提升付费转化 |
| 🟢 P2 | 2.2 数据信任页 | 建长期信任 |
| 🟢 P2 | 3.2 首页转化 | 打磨细节 |

## 不做的事

- ❌ UGC 论坛（教育王国 125 万会员，拼不过）
- ❌ 实时排名（HKPEP/SOSOMAMA 已垄断，没必要重复造轮子）
- ❌ 海外升学板块（Ohpama 已深耕，我们聚焦本地择校）
- ❌ 大规模网站重写/框架迁移（违背单一职责原则）
